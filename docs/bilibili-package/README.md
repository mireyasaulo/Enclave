# 隐界 B 站 7 天系列 · 视频生产交付包

> 收件人：manus（视频生成 AI agent）
> 委托方：项目维护者 / UP 主
> 目标：根据本包内文档，生产 7 条 B 站投稿视频（每集独立交付）。

---

## 1. 交付目标

| 项 | 内容 |
|---|---|
| 视频数量 | 7 条（编号 Day 1 ~ Day 7） |
| 平台 | bilibili.com |
| 分辨率 | 1920×1080（横屏，16:9），帧率 30fps，编码 H.264 mp4 |
| 单集时长 | 3~8 分钟（每集 `shot-list.md` 顶部有明确时长） |
| 音频 | 中文女声/男声皆可（保持 7 集同一人声），AAC 192kbps |
| 字幕 | 全程硬字幕（烧录），不要外挂 SRT |
| 封面 | 1146×717 JPG/PNG，每集 `cover-spec.md` 规定 |

每集独立产出一个交付文件夹（建议命名 `output/day-N/`），内含：

```
output/day-N/
├── video.mp4              # 主视频
├── cover.jpg              # 封面
├── publish-meta.txt       # B 站发布元数据（titel/desc/tags 已格式化）
└── pinned-comment.txt     # 置顶评论文案
```

---

## 2. 阅读顺序

按下面顺序读，再开始生产：

1. **`00-series-context.md`** — 项目是什么、调性是什么、不能说什么
2. **`00-visual-style.md`** — 片头/片尾/字幕/配色/字体的统一规范（7 集共享）
3. **`00-publish-schedule.md`** — 发布日历 + 跨集联动要点
4. 每一集进入 `day-N/` 目录后，依次读：
   - `script.md` — 旁白文稿（带时间码，直接 TTS）
   - `shot-list.md` — 镜头清单（每个镜头：素材来源/时长/转场/字幕）
   - `cover-spec.md` — 封面设计规格
   - `publish-meta.md` — 标题/简介/标签/置顶评论
5. 每集需要的素材集中在 `day-N/assets/` 与 `shared-assets/`

---

## 3. 素材使用约定

每个镜头在 `shot-list.md` 里都会标注素材来源，遵循以下三类：

- **🖼️ EXISTING** — 直接使用本包内的现成素材（路径会精确给出），不能用 AI 重画
- **🎨 GENERATE** — 提示词在镜头里给出，由 manus 用图像生成模型产出
- **📹 SCREEN-CAP-NEEDED** — 需要真实 app 录屏的段落（UP 主会另行提供，未到位时 manus 用 `assets/PLACEHOLDER-*.txt` 里的描述做临时占位图）

⚠️ **重要**：标 `📹 SCREEN-CAP-NEEDED` 的素材如果到生产时尚未提供，**不要自己用 AI 编造产品 UI**——会让真实用户识破。请用对应的 PLACEHOLDER 文本说明，把"待录屏"卡片留在视频里（带边框 + "此处录屏待补"字样），UP 主补录后会替换。

---

## 4. 旁白 / 配音规范

- 旁白文稿在 `day-N/script.md`，已分段并标注时间码
- 语速：**280-320 字/分钟**（B 站知识区典型节奏）
- 语气：**朋友式分享**，不是新闻播报，不是带货
- 不要加"大家好我是 XXX"开头——B 站系列视频从第 2 集开始观众已经认识 UP
- 不要加"点赞投币关注"统一收尾——CTA 在 `script.md` 末尾已写好定制版本

如果使用 TTS，推荐音色：
- 男声：阿里云 cosyvoice-v1（晓宇）/ 火山 zh_male_xiaoming
- 女声：阿里云 cosyvoice-v1（晓晨）/ 火山 zh_female_xiaomei
- 7 集必须用同一音色

---

## 5. 风险提示

- **不要在视频中念出 GitHub URL 完整字符串**——B 站对口播外链不友好。链接放置在简介+置顶评论
- **不要使用"震惊""逆天""颠覆""遥遥领先"等词**——B 站知识区观众反感，会降权
- **不要在画面里出现其他平台 logo**（YouTube/TikTok/X）——会触发审核延迟
- **不要在 Day 1-6 念出"求 star""求三连"**——只有 Day 7 收官集允许说一次
- BGM 必须使用 **B 站音频库** 或 **CC0 无版权** 曲目；推荐曲单在 `00-visual-style.md`

---

## 6. 交付检查清单

每集生产完成，自检以下项：

- [ ] 视频时长在 `shot-list.md` 给的范围内（±10 秒）
- [ ] 首 3 秒包含 `script.md` 标注的钩子原文，不能改写
- [ ] 全程字幕已烧录，字号 ≥ 60px（1080p 下移动端可读）
- [ ] 出现的产品截图未变形、未拉伸（保持原比例 + 留边/模糊背景填充）
- [ ] 封面与 `cover-spec.md` 描述一致
- [ ] `publish-meta.txt` 标题字数 ≤ 80 字符
- [ ] BGM 来自允许清单，不是流行歌
- [ ] 视频中不出现需打码的真实手机号/邮箱/IP

## 7. 联系与反馈

生产过程中如发现：
- 任何 PLACEHOLDER 素材描述不够清晰
- 任何镜头时间码与旁白对不上
- 任何"按当前世界状态"的指代（如"今天""这周"）需要 UP 主确认

→ 把疑问列在 `output/questions.md` 中，统一反馈给 UP 主，不要自行猜测后强行生产。
