```

- 整段背景音：一声短促的 UI 音效（"叮"或低频脉冲），≤ 0.8 秒

素材：
- logo：shared-assets/brand/logo.svg
- 字体：思源黑体 Heavy（中文）/ Inter Bold（英文）

```

⚠️ 这 2 秒是 B 站系列辨识度核心，**7 集必须完全一致**，包括音效。

## 3. 字幕规范（全程烧录硬字幕）

### 3.1 字号 / 位置 / 描边

```
字号：72px（1080p 下，移动端观看也清晰）
字体：思源黑体 Bold
颜色：#FFFFFF
描边：4px 黑色（#000000）外描边 + 60% 透明黑色背景条
位置：画面下方 1/6 处居中（距下边缘 120px）
最长每行字数：≤ 18 个中文字符（超长换行）
```

### 3.2 字幕节奏

- 每条字幕停留时长：**≥ 1.5 秒**（不论旁白多短）
- 不要"逐字弹出"动效——B 站知识区观众反感
- 简单的"硬切入 + 0.2s 淡出"即可
- 关键名词（如 "AI"、"开源"、"DeepSeek"、"一人一世界"）首次出现时加 **黄色（#FBBF24）高亮**

## 5. 产品截图的呈现方式

⚠️ 提供的产品截图都是 **390×844 手机竖屏比例**，与 16:9 横屏视频不匹配，必须做"画中画 + 衬底"处理：

### 方案 A：单图展示（覆盖整个画面）

```
画面布局：
- 全屏背景：手机截图自身的高斯模糊版本（blur ≥ 60px），整张铺满
- 中央：原图 1:1（不缩放、不变形），位于画面正中，向左偏 60px 留出右侧字幕空间
- 截图加 8px 圆角 + 阴影（box-shadow: 0 20px 60px rgba(0,0,0,0.5)）
- 截图周围加一圈"手机壳"线框（可选，#374151 1px 描边）
```

### 方案 B：双图对比（左右排列）

```
画面布局：
- 左右各放一张截图，间距 80px
- 顶部各加 16px 字幕"白天"/"凌晨"等对比标签
- 背景：纯深色（#111827）或低对比渐变
```

### 方案 C：滚动演示（视频化截图）

把静态截图用 After Effects / FFmpeg 做"上下滑动"动效，模拟用户在 app 内滚动浏览。每个截图滚动时长 3-5 秒。

## 6. 转场规范

### 允许使用的转场（4 种）

1. **硬切（cut）** —— 默认转场，70% 镜头之间使用
2. **0.2s 黑场（dip-to-black）** —— 章节卡前后使用
3. **0.3s 白闪（white flash）** —— 强调"惊喜""转折"瞬间，每集最多 2 次
4. **左右滑（slide L/R）** —— 仅用于"对比类"镜头切换

### 禁止使用

- ❌ 复杂的 3D 翻转、旋转、像素粉碎
- ❌ 心形 / 闪光 / 卡通弹出动画
- ❌ 任何"Premiere/剪映模板预设"的炫技转场

B 站知识区观众喜欢"克制"——简单转场比花哨更显专业。

## 7. BGM 规范

### 7.1 允许使用的音源

只能从以下三处选曲：

1. **B 站音频库**（创作中心 → 音频库 → 站方授权）
2. **YouTube Audio Library**（标注 "No attribution required"）
3. **Free Music Archive（FMA）**（CC0 / CC-BY 协议）

### 7.2 各集 BGM 风格指引

| 集 | 风格 | 推荐曲风 | 音量 |
|---|---|---|---|
| Day 1 开篇 | 神秘+期待 | Ambient electronic / Lo-fi tech | -22dB |
| Day 2 奇观 | 节奏紧凑 | Upbeat indie pop / Lo-fi hip-hop | -20dB |
| Day 3 设计 | 沉静温暖 | Lo-fi study beats | -24dB |
| Day 4 教程 | 中性背景 | Minimal piano / Soft ambient | -26dB（教程要让命令音清楚） |
| Day 5 观点 | 深度+情感 | Cinematic strings light / Post-rock 慢段 | -22dB |
| Day 6 抖音风 | 活泼有节奏 | Funky electronic / Future bass 轻量版 | -20dB |
| Day 7 收官 | 收束感动 | Piano ballad / Acoustic guitar | -22dB |

### 7.3 BGM 处理规则

- **不要使用流行歌**（即使有授权，也会让视频显得"业余")
- 旁白说话时 BGM 自动降到 -28dB（自动 ducking）
- 旁白间隙 BGM 回升到正常音量
- 视频前后 3 秒做淡入淡出

## 9. 镜头时长建议（防止"信息过载"或"节奏拖沓"）

| 镜头类型 | 推荐时长 |
|---|---|
| 单条文字字幕的镜头 | 2-4 秒 |
| 静态产品截图（无动效） | 3-5 秒 |
| 动态演示（滚动/滑动） | 5-8 秒 |
| 章节卡 | 1.0 秒（固定） |
| 片头/片尾 | 2-3 秒（固定） |
| 钩子段镜头 | ≤ 1.5 秒/镜（快速切换出冲击力） |

任何超过 **8 秒**的单镜头，必须有至少一次"画面内变化"（缩放/平移/字幕弹出/数字变化），否则观众会跳走。
