# Day 4 需录屏的镜头（占位说明）

> Day 4 是教程集，录屏比例最高（11 段），必须完整且无错。

---

## 录屏 1：终端 clone + cd

**对应镜头**：镜头 007

**命令**：
```
git clone https://github.com/yuanzui0728/enclave.git
cd enclave
ls
```

**规格**：
- 终端字体 ≥ 24pt
- 配色：浅色背景（不要纯黑——B 站移动端看不清）或深底加亮字
- 文件名：`day-4-cap-01-clone.mp4`，约 25 秒

---

## 录屏 2：cp + 编辑 .env

**对应镜头**：镜头 008

**命令**：
```
cp api/.env.example api/.env
nano api/.env  # 或 vim / VSCode
```

编辑界面里要看到 `DEEPSEEK_API_KEY=` 和 `ADMIN_SECRET=` 两行，**圈红**两个字段

**敏感信息**：录屏时**用假 KEY**（如 `sk-xxxxxxxxxxxxx`），不要露真实 KEY

**文件名**：`day-4-cap-02-env-edit.mp4`，约 20 秒

---

## 录屏 3：docker compose up -d（含加速）

**对应镜头**：镜头 010

**命令**：
```
docker compose up -d
```

**录制要求**：
- 完整录第一次启动（镜像下载 + 容器启动，约 1-2 分钟）
- 剪辑时把镜像下载段做 2 倍加速，**屏幕角标加"已加速 2x"**
- 容器启动完成的最后输出保持原速

**文件名**：`day-4-cap-03-compose-up.mp4`，原始 90 秒 → 剪辑后 30 秒

---

## 录屏 4：docker compose ps

**对应镜头**：镜头 011

**命令**：

**文件名**：`day-4-cap-04-compose-ps.mp4` 或 `.png`

## 通用注意事项（所有录屏）

1. **不要露任何真实 KEY / 邮箱 / IP**——录屏前把环境变量改成假数据
2. **命令出现的时候要"停留"几秒**——给观众抄写时间
3. **错误不要剪掉**——如果第一次没起来，把错误段保留 + 念出来解决方案。"真实有用"比"假装顺利"加分
4. **加速段必须明确标识**——画面右上角加 "已加速 2x" 字样
5. **所有录屏 1080p 30fps**——不要 4K，浪费容量
