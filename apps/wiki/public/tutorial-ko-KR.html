Enclave Wiki · 입문 튜토리얼

가입·탐색부터 AI 보조 작성, 순찰 심사, 관리자 도구까지 — 이 한 페이지로 전체 기능을 안내합니다.

0환영합니다

Enclave Wiki는 AI 캐릭터 창작을 위한 협업 백과사전입니다. 누구나 열람할 수 있고, 가입하면 캐릭터 문서를 작성·편집할 수 있습니다. 여기에 적는 내용은 단순한 설명문이 아니라, AI가 그 캐릭터를 직접 "연기"하는 데 사용하는 구조화된 캐릭터 레시피(recipe)입니다.

운영 흐름은 위키백과와 매우 비슷합니다:

  • 모든 콘텐츠는 협업으로 편집 가능하며, 모든 변경에 버전 이력이 남습니다
  • 신규 사용자의 편집은 먼저 심사 대기열에 들어가고, 순찰자가 승인해야 공개됩니다
  • 차이 비교(diff), 되돌리기(revert), 보호(protection), 신고(report) 등 협업의 표준 도구가 모두 갖춰져 있습니다

위키백과와 다른 점은 문서 자체가 AI의 "영혼"이라는 것입니다. 입력하는 이름·소개·성격·전문 영역·대화 시나리오 같은 필드는 구조화된 레시피로 묶여 대규모 언어 모델에 입력됩니다. 문서 품질이 곧 AI 성능을 결정합니다.

💡 팁

어디부터 시작할지 모르겠다면 다음 절 "5분 빠른 시작"을 따라 직접 한 번 해보세요. 모든 개념이 자연스럽게 연결됩니다.

15분 빠른 시작 사용자

1

계정 가입

오른쪽 상단 "가입" 클릭, 사용자명(2자 이상)과 비밀번호(6자 이상)를 입력. 신규 계정은 newcomer 등급에서 시작합니다.

2

캐릭터 탐색

홈으로 돌아와 카드를 클릭해 보세요 — 완성된 문서의 모습을 직접 확인할 수 있습니다.

3

검색 시도

상단 검색창에 키워드를 입력하고 Enter. 이름·관계·소개·성격 4개 필드를 함께 검색합니다.

4

비공개 캐릭터 만들기

사이드바 → "내 비공개 캐릭터" → "만들기". 최소 정보(이름 + 소개 + 관계)를 채운 후 AI 생성으로 나머지 필드를 보완.

5

문서 편집 시도

공개 문서를 열고 "편집" 클릭, 작은 부분을 수정하고 10자 이상의 편집 요약을 남겨 제출. 순찰자 심사를 기다립니다.

💡 팁

신규 기간 동안 공개 문서 편집은 모두 심사를 거칩니다. 일정량의 편집을 쌓고 자동으로 autoconfirmed로 승급되면 저위험 편집은 즉시 반영됩니다.

2가입 & 로그인 사용자

가입

/register에 접속해 입력:

  • 사용자명: 2자 이상, 공개되며 편집 이력에 표시됩니다
  • 비밀번호: 6자 이상, 본인만 볼 수 있음
  • 비밀번호 확인: 위와 동일

가입 후 계정은 신규 등급(newcomer)으로 설정됩니다. 즉시 편집할 수 있지만 공개 문서 편집은 심사 대기열을 거칩니다.

로그인

/login에서 두 가지 방법:

  • 사용자명 + 비밀번호: 표준 방법
  • 이메일 인증 코드: 등록 이메일로 코드 발송, 입력하면 로그인 — 비밀번호를 잊었을 때 유용

로그인이 필요한 페이지에 비로그인 상태로 접근하면 /login?redirect=...으로 이동되며, 로그인 후 자동으로 원래 페이지로 돌아갑니다.

계정 설정

오른쪽 상단 사용자명 → "계정 설정"에서:

  • 비밀번호 변경(이메일 코드 확인 필요)
  • 이메일 확인 / 변경
  • 프로필 정보 확인: 현재 등급(newcomer/autoconfirmed/patroller/admin), 누적 편집 수, 승인 편집 수, 되돌림 횟수, 자동 확인 시점
⚠️ 주의

이메일은 비밀번호 복구의 유일한 수단입니다. 가입 후 즉시 등록하세요. 이메일이 없으면 비밀번호 분실 시 자가 복구가 불가하며 관리자 도움이 필요합니다.

3문서 탐색 사용자

캐릭터 디렉터리(홈)

홈 /에는 모든 공개 캐릭터가 인기도 / 최근 활동 순으로 표시됩니다. 카드마다 아바타, 이름, 관계 유형, 소개 첫 줄이 표시됩니다.

캐릭터 페이지 구성

카드를 열면:

  • 헤더: 아바타, 이름, 관계 유형 배지(친구 / 가족 / 멘토 / 전문가 / 사용자 정의), 라이프사이클 배지(active / pending create / deleted / pending delete)
  • 본문 탭:
    • "내용": 소개, 성격, 전문 영역, 트리거 시나리오, 관계 설명
    • "이력": 모든 리비전과 차이 비교
    • "토론": 이 문서 관련 토론 스레드
  • 오른쪽 작업: ⭐ 관심 / 해제, ✏️ 편집(로그인 필요), 🗑️ 삭제 신청(로그인 필요), 보호 상태 표시

├──────────────────────────────────────────────────────────────┤
│ [ 내용 ] [ 이력 ] [ 토론 ]                                    │

└──────────────────────────────────────────────────────────────┘

안정 버전 vs 현재 버전(stable / current)

같은 문서에 두 가지 뷰가 있을 수 있습니다:

  • 안정(stable): 모든 사용자가 기본으로 보는 버전 — 가장 최근에 승인된 내용
  • 현재(current): 미심사 편집을 포함한 버전 — 편집자 본인과 순찰자만 볼 수 있음

즉, 편집 직후 본인은 변경 사항을 볼 수 있지만(current), 다른 방문자는 승인 전까지 이전 stable 버전을 봅니다.

4검색 사용자

모든 페이지 상단 검색창에 입력 후 Enter하면 /search?q=키워드로 이동합니다.

검색 범위

백엔드는 다음 필드를 전문 인덱싱합니다:

  • 캐릭터 이름
  • 관계 설명("가장 친한 친구" 등)
  • 소개(bio)
  • 성격(personality)

결과 정렬

관련도 점수 순으로 정렬되며, 각 결과에 0–100의 관련도 백분위가 표시됩니다. 클릭하면 캐릭터 페이지로 이동합니다.

💡 팁

비로그인 사용자도 검색이 가능합니다 — 외부 공개 기능이라 시험 트래픽에 활용할 수 있습니다.

5문서 작성 & 편집 사용자

본 위키의 핵심 기능입니다. 모든 공개 캐릭터는 동일한 폼으로 작성·관리됩니다.

5.1 새 문서 만들기

사이드바 → "캐릭터 만들기", 또는 /create. 폼은 6개 섹션:

섹션주요 필드설명
① 기본 정보이름, 아바타, 소개, 관계 유형, 관계 설명이름과 소개는 필수 — 다른 사람이 캐릭터를 식별하는 기본 정보
② 대화 스타일말투, 입버릇, 감정 톤AI의 어투를 결정
③ 핵심 로직가치관, 결정 패턴, 금기캐릭터의 "두뇌" — 갈등 시 사고 방식
④ 트리거 시나리오대표 대화 오프닝, 롤플레이 상황이 캐릭터가 호출되는 상황
⑤ 기억 설정배경, 주요 경험캐릭터가 "기억하는" 것
⑥ 소셜 파라미터전문 영역, 상호작용 선호추천 및 검색 매칭에 영향

5.2 AI 원클릭 생성

각 섹션 우상단에 "AI 생성" 버튼. 대규모 모델을 호출해 빈 필드를 채워줍니다.

💡 권장 워크플로

먼저 이름 + 소개 + 관계 설명(3개의 앵커)을 직접 입력 → "전체 생성" 클릭. AI가 이 3개를 기반으로 다른 모든 빈 필드를 채웁니다. 이미 작성된 내용은 덮어쓰지 않고 빈 필드만 보완합니다.

주요 매개변수:

  • 속도 제한: 사용자당 시간당 15회(비공개 + Wiki 편집 공유 버킷)
  • "최적화" 모드(optimize): 활성화 시 기존 내용도 덮어쓰기 — 작성한 내용을 AI에게 다듬게 하고 싶을 때
  • "전체 생성" 전제 조건: name + bio + relationship가 채워져 있어야 함, 그렇지 않으면 버튼이 비활성화
  • 임시 저장 자동화: 매 생성마다 임시저장이 남으므로 페이지를 닫아도 잃지 않음

5.3 임시 저장 시스템

편집 중 폼이 자동 저장됩니다:

  • 브라우저 측: 실시간(닫고 다시 와도 남음)
  • 서버 측: AI 생성 또는 수동 저장 시 스냅샷
  • 복원 방법: /my-drafts → 해당 임시저장 클릭 → ?draftId=... 포함된 편집 페이지로 복귀

5.4 편집 제출

"제출" 버튼 전에 입력해야 할 항목:

  • 편집 요약(editSummary): 변경 내용 간단 설명. 10자 이상이 라이프사이클 변경 등 고위험 편집의 필수 요건
  • 사소 편집 표시(isMinor): 오타·서식·공백 등 사소한 변경이면 체크 — 순찰자 처리 속도가 빨라짐
⚠️ 편집 충돌 감지

편집 중 다른 사람이 같은 문서를 수정·제출한 경우, 제출 시 충돌이 감지됩니다. baseRevisionId로 판정하며, 최신 버전을 다시 열어 수동 병합해야 합니다.

5.5 제출 후 동작

당신의 등급에 따라 다릅니다:

등급저위험 편집고위험 편집
newcomer심사 대기열로심사 대기열로
autoconfirmed즉시 반영심사 대기열로(여전히 10자 이상 요약 필요)
patroller / admin즉시 반영즉시 반영

5.6 기존 문서 편집

문서 페이지 오른쪽 ✏️ "편집" 버튼 → /character/:id/edit로 이동. 폼에는 최신 버전(미심사 필드 포함, 본인만 표시)이 미리 채워져 있고, 나머지는 만들기와 동일한 흐름입니다.

6내 비공개 캐릭터 사용자

공개 Wiki 문서와는 별도로, 로그인한 사용자는 본인만 볼 수 있는 전용 공간 「내 비공개 캐릭터」(/my-characters)를 가집니다. 자유롭게 만들고 실험하고 백업할 수 있으며, 모든 작업은 즉시 반영됩니다. 심사가 필요 없고, 다른 사람에게 보이지 않으며, 검색 / 캐릭터 디렉터리 / 추천 목록에도 나타나지 않습니다.

6.1 Wiki 문서와의 차이

특성Wiki 문서비공개 캐릭터
가시성전체 공개본인만
심사 필요신규 사용자는 필요불필요
버전 이력완전한 이력 + diff없음 (createdAt / updatedAt 타임스탬프뿐)
순찰 / 되돌리기있음없음
토론 페이지있음없음
삭제소프트 삭제(복구 신청 가능)하드 삭제(되돌릴 수 없음)
폼 필드personality / editSummary / isMinor 포함personality 등 없음, 더 가벼움
용도공공 라이브러리 구축개인 창작 / 시험 / 백업

6.2 진입점

  • 목록: 사이드바 → 「내 비공개 캐릭터」(/my-characters)
  • 새로 만들기: 목록 우상단 「✨ 새 캐릭터」 → /my-characters/new
  • 편집: 목록에서 임의의 카드 클릭 → /my-characters/:id
  • 가져오기: 목록 우상단 「📥 파일 가져오기」 → .json 선택

6.3 폼 필드 (6 섹션)

섹션주요 필드필수?
① 기본 정보이름, 아바타(emoji / URL), 관계 설명, 관계 유형, 전문 영역, 소개이름 + 관계 설명 필수
② 대화 답변대화 시나리오 프롬프트아니오 (권장)
③ 핵심 로직핵심 로직, 망각 곡선 (0–100, 기본 70, 고급)아니오 (권장)
④ 시나리오 프롬프트모먼츠 게시, 광장 게시, 비디오 채널 게시, 모먼츠 댓글, 광장 댓글, 친구 요청 / 흔들기 인사, 능동 알림 (총 7개)아니오
⑤ 기억 프롬프트최근 기억, 장기 기억 (고급)아니오
⑥ 소셜 파라미터소셜 개방도(open / normal / private), 능동 탐색 확률(0–1, 기본 0.3), 친밀도 시드(0–100, 기본 0)기본값 있음
💡 Wiki 편집 폼과의 차이

비공개 캐릭터는 personality / 트리거 시나리오 / editSummary / isMinor 필드가 없어 더 간결합니다. 또한 버전 이력이 없으므로 baseRevisionId 충돌 감지도 없습니다(「마지막 수정 시각」만 존재).

전문 영역(총 개수)50개
recipe JSON / profile JSON각 64 KB

6.5 AI 원클릭 생성

각 섹션 우상단에 ✨ AI 생성 버튼; 페이지 상단에 ✨ AI로 전체 생성 버튼.

  • 생성 가능한 5개 섹션: basics(아바타 / 전문 영역 / 관계 유형), core_logic(핵심 로직 / 망각 곡선), chat, scenes(7개 시나리오 프롬프트), memory(최근 / 장기 기억)
  • 「전체 생성」 전제 조건: name + bio + relationship 세 sacred 필드를 반드시 먼저 입력. 미입력 시 백엔드가 400으로 거부
  • optimize 모드: 기본은 빈 필드만 보완하며 기존 내용을 덮어쓰지 않음. 이미 내용이 있는 섹션에서 AI 생성을 누르면 확인 다이얼로그가 뜨며, 확인 후 덮어씀
  • 생성 중: 해당 섹션의 모든 필드가 disabled 상태가 되며 「✨ AI가 생성 중…」 표시
⚠️ 속도 제한

AI 생성 전용 할당량: 사용자당 시간당 15회(Wiki 문서 편집과 같은 버킷 공유). 초과 시 WIKI_AI_RATE_LIMITED 반환, 「AI 생성 할당량 소진: 시간당 15회 한도, X초 후 재시도」 안내.

CRUD(생성 / 수정 / 삭제 / 가져오기) 별도 할당량: 사용자당 시간당 60회.

6.6 임시 저장 시스템

「새로 만들기 페이지 + 전체 생성」 경로에서만, 백엔드가 자동으로 character_drafts에 임시 저장:

  • 페이지나 브라우저를 닫고 다시 와도 /my-drafts 목록에서 찾을 수 있음
  • 임시 저장 항목 클릭 → ?draftId=... 파라미터로 새로 만들기 페이지로 복원
  • 복원 시 에디터에 미저장 내용이 있으면 덮어쓸지 묻는 confirm 다이얼로그 표시

6.7 JSON 가져오기 / 내보내기

내보내기

  • 목록의 각 카드 「📤 내보내기」 / 편집 페이지 우상단 「📤 JSON 내보내기」
  • 파일명: 캐릭터 이름(잘못된 문자 치환) + .json
  • 스키마: "$schema": "yinjie-private-character/v1"
  • 포함 필드: name / avatar / bio / relationship / relationshipType / expertDomains / recipe / profile / 소셜 파라미터 + meta(exportedAt, exportedBy, version)
  • 제외: isOnline / isTemplate / sourceType / aiRelationships 등 관리자 전용 필드

가져오기

  • 목록 우상단 「📥 파일 가져오기」로 .json 선택
  • 파일 크기 상한: 5 MB
  • 캐릭터 이름으로 upsert: 기존 캐릭터와 동명 → 덮어쓰기, 다른 이름 → 신규 생성
  • 토스트로 덮어쓰기 / 신규 구분: 「동명 캐릭터를 덮어썼습니다: X」 vs 「새 캐릭터를 가져왔습니다: X」
  • 일괄 가져오기 미지원(1 파일 = 1 캐릭터)
  • 스키마 버전 불일치 / 필드 초과 → 400 거부

6.8 은계 app에 연동해 실제로 사용하기

wiki 쪽 비공개 캐릭터는 「설정 파일」일 뿐입니다. AI가 실제로 그 캐릭터를 연기하며 대화하게 하려면, 은계 app(런타임 클라이언트)으로 가져와야 합니다. 전체 흐름:

1

wiki에서 내보내기

비공개 캐릭터 목록 / 편집 페이지에서 「📤 JSON 내보내기」 클릭 → .json 다운로드.

2

은계 app 열기

임의의 클라이언트(모바일 / 데스크톱)에서 로그인. 가져오기는 로그인 필요.

3

가져오기 페이지로 이동

모바일: 「나」 탭 → 「캐릭터 가져오기」.
데스크톱: 메뉴 진입점 없음, /profile/character-import 직접 접속.

4

.json 업로드

방금 다운받은 파일(≤ 5 MB) 선택 → 제출. 백엔드가 POST /characters/import-personal로 검증·저장.

5

대화 시작

가져오기 성공 후 캐릭터는 자동으로 친구 추가되어 「연락처」에 표시. 연락처에서 캐릭터 탭 → 캐릭터 상세 페이지 → 「대화 시작」으로 채팅 시작.

⚠️ 버전 동기화 없음

wiki 쪽과 app 쪽은 독립된 사본입니다. wiki에서 캐릭터 설정을 수정해도 app은 자동 업데이트되지 않으며, app에서 대화로 쌓인 기억 / 관계 진행도는 wiki에서 보이지 않습니다.

동기화하려면: wiki에서 다시 내보내기 → app 가져오기 페이지에서 재업로드. 동명 덮어쓰기는 이전에 가져온 캐릭터에만 적용(sourceType=private_import) — 다른 출처의 동명 캐릭터는 충돌 에러로 덮어쓸 수 없습니다.

💡 왜 「wiki에서 만들고 app에서 실행」?

app에는 「비공개 캐릭터 만들기」 UI가 없습니다. 설계상 wiki는 「완전한 캐릭터 편집 환경 + AI 보조」, app은 「대화 런타임」 — 만들기·편집은 wiki, 실행은 app. 이 분리가 wiki에 비공개 캐릭터 기능이 존재하는 핵심 이유입니다.

6.9 삭제

⚠️ 하드 삭제, 복구 불가

비공개 캐릭터 삭제는 실삭제(Wiki 문서의 소프트 삭제와 다름) — 확인 즉시 DB에서 완전히 사라지며, 「복구 신청」 절차가 없습니다. 삭제 전에 「『X』를 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다.」 확인 다이얼로그가 표시됩니다.

장기 보관이 필요하면, 먼저 「📤 JSON 내보내기」로 로컬에 저장한 후 삭제하세요.

주의: wiki 쪽에서 삭제해도 이미 app에 가져온 사본에는 영향이 없습니다(둘은 독립) — app 쪽에서도 없애려면 연락처에서 별도로 삭제하세요.

6.10 현재 미지원

비공개 캐릭터에서 현재 할 수 없는 것을 알아두면, 진입점을 찾느라 헤매지 않습니다:

  • ❌ wiki 안에서 직접 캐릭터와 대화 불가: wiki 쪽에 대화 UI가 없습니다 — 6.8에 따라 app으로 가져오세요
  • ❌ 공유 / 협업 불가: 다른 사용자에게는 로그인해도 접근 불가
  • ❌ 「Wiki에 일괄 공개」 기능 없음: 비공개 캐릭터를 공개 문서로 만들려면 /create에서 직접 다시 입력
  • ❌ 버전 이력 없음: 저장 시마다 덮어써지며, 이전 버전은 볼 수 없음
  • ❌ 일괄 / 정렬 / 필터 / 검색 없음: 목록은 시간 역순, 많아지면 스크롤로 찾아야 함
💡 권장 워크플로

비공개 캐릭터는 당신의 「초안 모래판」: 심사 부담 없이 자유롭게 시도할 수 있고 다른 사람에게도 영향 없음. 마음에 드는 캐릭터가 완성되면, 핵심 필드(이름, 소개, 관계, 주요 프롬프트)를 「공개 문서 만들기」에 옮겨 적어 심사 제출 — 현재 비공개 → 공개의 유일한 경로입니다.

7이력 & 차이 사용자

모든 공개 문서의 모든 변경은 영구 보존되며, 임의의 과거 버전을 추적할 수 있습니다.

이력 탭

문서 페이지 → "이력" 탭 → 시간 역순 리비전 목록. 각 행:

  • 리비전 ID
  • 편집자 사용자명
  • 타임스탬프
  • 편집 요약
  • 작업 유형(생성 / 편집 / 삭제 신청 / 복구 신청 / 되돌리기)
  • 위험 등급(저 / 중 / 고)
  • 순찰 완료 여부 / 심사 완료 여부

차이 비교(diff)

임의의 두 리비전 선택 → "비교" → /character/:id/diff?from=A&to=B로 이동, 좌우 병렬 표시:

  • 변경된 필드(diffFromParent)
  • 각 변경 필드의 이전 vs 새 값
  • 레시피 JSON 구조 차이(접기 가능)
📐 ASCII 다이어그램: diff 페이지 레이아웃 
┌─────────────────────┬─────────────────────┐
│ 이전(v12)           │ 새(v13)             │
├─────────────────────┼─────────────────────┤
│ 소개                │ 소개                │
│ 다정한 친구.        │ 다정하고 수다스러운 │
│                     │ 친구.               │
├─────────────────────┼─────────────────────┤
│ 성격                │ 성격                │
│ 내성적, 조용함      │ 내성적이지만        │
│                     │ 좋은 청자           │
└─────────────────────┴─────────────────────┘

diff 페이지는 로그인 없이 누구나 열람 가능 — 투명한 개정 이력은 위키 협업의 기반입니다.

8관심 목록 & 최근 변경 사용자

8.1 관심 등록(Watch)

캐릭터 페이지 우상단 ⭐ 버튼으로 토글. 관심 등록 후:

  • 해당 문서의 새 편집이 "내 관심 목록"에 표시됨
  • 캐릭터별 알림 설정 가능: 편집 알림(notifyOnEdit), 토론 알림(notifyOnTalk)

8.2 내 관심 목록

사이드바 → "내 관심 목록"(/watchlist). 2단 레이아웃:

  • 왼쪽: 관심 등록한 모든 캐릭터, 각 행에 "해제" 버튼; 삭제된 항목은 삭제됨 배지
  • 오른쪽: 모든 관심 캐릭터의 최근 활동 통합 피드 — 편집 리비전과 토론 신규 게시글 모두

8.3 최근 변경(Recent Changes)

사이드바 → "최근 변경"(/recent-changes) — 사이트 전체 타임라인, 누구나 열람 가능:

  • 모든 문서의 모든 리비전(관심 범위 외 포함)
  • 라이프사이클 신청(삭제 / 복구) 및 되돌리기 포함
  • 각 행에 위험 등급, 작업 유형, 순찰 상태 표시
💡 순찰자용

순찰자 시점에는 "미순찰만" 필터가 추가됩니다 — 일상 순찰 작업에 사용.

9토론 페이지(Talk) 사용자

모든 문서에는 독립된 토론 페이지가 있어, 위키백과의 "talk page"처럼 "이 캐릭터를 어떻게 작성할 것인가"를 논의합니다(캐릭터와 대화하는 곳이 아닙니다).

주요 작업

  • 새 토론 글: 제목 + 본문, 새 주제 시작
  • 답글: 임의의 게시글에 답글(parentPostId로 중첩)
  • 본인 게시글 삭제: 소프트 삭제(기록 보존, 내용 비표시)
  • 게시글 신고: 신고 절 참조

순찰자 가능 작업

  • 스레드 잠금(isLocked): 새 답글 금지 — 토론이 격해질 때
  • 해결됨 표시(isResolved): 합의에 도달한 주제

10콘텐츠 신고 사용자

로그인 사용자라면 누구나 문제 콘텐츠를 신고할 수 있습니다. 대상은 3가지:

  • 문서 리비전(wiki_revision): 어떤 편집이 문제를 야기(반달리즘 / 허위 정보 / 장난)
  • 토론 게시글(wiki_talk_post): 토론 중 인신공격 / 스팸
  • 문서 페이지(wiki_page): 애초에 존재해서는 안 되는 캐릭터(저작권 / 중대 위반)

신고 절차

  1. 대상 옆 "🚩 신고" 버튼 찾기
  2. 사유 선택(스팸 / 허위 / 악용 / 기타)
  3. 선택적으로 상세 설명 입력(500자 이하)
  4. 제출

신고는 자동으로 관리자 /admin/reports 큐에 들어가 사람이 처리합니다. 결과는 3가지:

  • 해결됨(resolved): 관리자가 처리(삭제 / 되돌리기 / 차단 등)
  • 기각(dismissed): 신고가 성립하지 않는다고 판단
  • 미처리(open): 처리 대기
⚠️ 주의

악의적 신고도 처분 대상 — 무효 신고를 반복하면 관리자 주의를 끌게 됩니다.

12순찰자 핸드북 순찰자

순찰자는 플랫폼 일상 품질의 첫 번째 방어선입니다. 처리 업무는 3가지: 미공개 편집 심사, 공개 편집 순찰, 잘못된 변경 되돌리기.

12.1 심사 대기열

사이드바 → "심사 대기"(/pending-reviews, 순찰자 이상만 표시).

큐 항목:

  • 제출자 / 제출 시각 / 대상 문서
  • 작업 유형(생성 / 편집 / 삭제 / 복구 / 되돌리기)
  • 위험 등급(저 / 중 / 고 — 고는 10자 이상 요약 필요)
  • 리비전 종류
  • 편집 요약
  • 차이 미리보기 링크

처리 옵션:

작업효과
승인(approve)편집이 안정 버전에 병합되어 모든 사용자에게 공개
기각(reject)편집 폐기, 최신 버전에 자동 동기화 안 됨(제출자가 재편집 필요)
수정 요청(request changes)코멘트 남겨 제출자에게 수정 요청
💡 순찰 페이스 팁

"고위험 + 긴 편집 요약" 제출을 우선 처리(대개 진지한 큰 변경). 저위험 작은 수정은 일괄로 빠르게 통과.

12.2 "최근 변경" 순찰(Patrol)

"순찰"과 "승인"은 다른 일입니다:

  • 승인(review approval): 미공개 편집을 발효 — newcomer 제출 대상
  • 순찰(patrol): 이미 발효된 편집을 "확인했음, 문제없음"으로 표시 — autoconfirmed 제출 대상(이미 자동 발효되었지만 확인 필요)

/recent-changes에서 "미순찰만" 필터로 전환, 한 줄씩 확인, 문제없으면 "순찰됨 표시" 클릭.

12.3 되돌리기(Revert)

이력 탭에서 과거 버전 옆 "이 버전으로 되돌리기" 버튼:

  1. 클릭 → 되돌리기 사유 입력 모달
  2. 제출 → 새 리비전 생성(operation: "revert")으로 내용을 대상까지 되돌림
  3. 순찰자/관리자라면 즉시 발효, 그렇지 않으면 심사 대기열(이론상 비순찰자에게 이 경로는 노출되지 않음)
⚠️ 주의

되돌리기는 제출자의 revertedCount에 기록되어 향후 승급에 영향을 미칩니다. 명백히 악의적·해로운 편집인 경우에만 실행하세요. 의견 차이는 토론 페이지에서.

13관리자 핸드북 관리자

🛡️ 관리자 전용

본 절의 /admin/* 페이지는 비관리자 계정에 404를 반환합니다.

13.1 사용자 & 권한 관리

/admin/users 접근. 기능:

  • 전 사이트 사용자 목록 조회(사용자명, 유형, 등급, 가입일, 편집 통계)
  • 임의 사용자의 등급 변경(newcomer ↔ autoconfirmed ↔ patroller ↔ admin)
  • 각 사용자의 누적 편집 수, 승인 수, 되돌림 수, 순찰 수, 마지막 편집 시각, 자동 확인 시각 조회

본인 등급은 변경 불가(실수로 자신을 백오피스에서 잠그는 것을 방지).

13.2 차단(Block)

/admin/blocks 접근. 사용자 특정 작업 방지:

scope효과
global전 사이트 침묵: 편집 / 토론 / 신고 모두 불가
page지정 문서 편집만 금지
talk토론 게시만 금지(편집은 가능)

각 차단 레코드: 사용자 / scope / 사유 / 생성자 / 생성 시각 / 만료(expiresAt, 비우면 영구) / 해제자 / 해제 시각. 목록은 "활성"과 "만료/해제됨"으로 분할.

13.3 페이지 보호(Protection)

/admin/protection 접근. 단일 문서에 편집 장벽 추가:

페이지 레벨 보호

level편집 가능자
none모든 로그인 사용자(기본)
semiautoconfirmed 이상 — 신규 차단
fullpatroller 이상 — 신뢰 사용자만

심사 정책(reviewPolicy)

  • open: 편집 즉시 발효, 강제 심사 없음
  • pending_changes: 편집 모두 보류, 승인 필요

필드 레벨 보호

페이지 내 특정 필드(예: name)에 최소 편집 등급 별도 설정:

fieldPath: "name"
minRoleToEdit: "patroller"

용도: 악의 수정 위험이 높은 필드(캐릭터 이름, 관계 유형)만 잠그고 다른 필드는 평소대로 협업 유지.

모든 보호 변경은 로그화: oldLevel → newLevel + 변경자 + 사유 + 만료, 추적 가능.

13.4 신고 큐

/admin/reports 접근. 사용자 신고 처리:

  • 상태별 그룹: 미처리(open) / 해결됨(resolved) / 기각(dismissed)
  • 각 항목: 신고자, 대상 유형(리비전 / 게시글 / 페이지), 사유, 상세, 시각
  • 처리: 상태를 resolved 또는 dismissed로 변경(실제 조치 — 삭제, 되돌리기, 차단 — 은 다른 백오피스에서 별도 수행)

13.5 반파괴 필터(Abuse Filter)

/admin/abuse-filters 접근. 정규식 기반 콘텐츠 차단 규칙:

필드설명
name규칙명(백오피스 전용)
pattern정규 표현식
scope적용 범위(편집 / 토론 / 전부)
actionflag(이후 수동 심사) / reject(제출 거부) / log(기록만)
severitylow / medium / high — 후속 우선순위에 영향

규칙별 적중 통계도 동시 표시: 각 규칙 적중 횟수, 적중 사용자 분포, 최근 N건 적중 상세.

💡 정규식 팁

새 규칙은 먼저 action: log로 일주일 운용하며 적중 범위를 확인, 오탐이 없으면 flag 또는 reject로 승격.

13.6 거버넌스 대시보드

/admin/wiki-stats 접근. 일원화된 개관:

  • 실시간 카운터: 오늘 신규, 이번 주 신규, 심사 대기열 길이
  • 사용자 품질 순위: 최다 되돌림, 저승인율, 주의 사용자
  • 일일 추세: 생성, 승인, 기각, 필터 적중, 자동 확인 승급
  • 필터 성능: 규칙별 적중률, 오탐률(수동 취소된 횟수)

14자주 묻는 질문(FAQ)

Q: 제출한 편집은 어디에? 왜 페이지가 그대로?

신규 사용자 편집은 심사 대기열로 갑니다. 본인은 변경을 볼 수 있지만(current 뷰), 다른 방문자는 승인된 stable 버전을 봅니다. 순찰자가 승인하면 모두에게 반영. /recent-changes에서 본인 제출 기록을 확인할 수 있습니다.

Q: AI 생성 버튼이 회색입니다

3가지 가능성:

  • "전체 생성"은 name + bio + relationship 입력이 선행되어야 함
  • 현재 시간 할당량 소진(사용자당 시간당 15회, 비공개+Wiki 공유)
  • 모든 필드가 이미 채워짐이고 최적화 모드 미선택 — 기본은 빈 필드만 보완

Q: 저와 다른 사람의 화면이 다른 이유?

본인은 current 뷰(본인 또는 다른 사람의 미심사 변경 포함)를 보고 있고, 비로그인 또는 편집에 미참여인 사용자는 stable 뷰(최근 승인 버전)를 봅니다. 설계상의 격리 — 미심사 콘텐츠는 공개에 노출하지 않음.

Q: autoconfirmed로 승급 방법?

고품질 편집을 지속적으로 — 시스템이 "승인 편집 수 + 가입 기간 + 낮은 되돌림율" 기준으로 자동 승급. 신청 불필요. 계정 설정에서 현 편집 통계 확인 가능.

Q: 비공개 캐릭터를 다른 사람이 볼 수 있나요?

볼 수 없습니다. 비공개 캐릭터는 /my-characters 목록에만 표시되며 관리자 백오피스에도 표시되지 않습니다. "Wiki에 동기화"를 능동적으로 수행하지 않는 한 공개되지 않습니다.

Q: 편집 충돌이 발생하면?

제출 시 시스템이 감지하여 경고를 표시. 가장 간단한 방법: 임시저장을 클립보드에 복사 → 편집 페이지 닫기 → 다시 열기(최신 버전 확보) → 본인 변경을 수동 병합 → 재제출.

Q: 비밀번호를 잊었을 때?

로그인 페이지에서 "이메일 인증 코드" 선택 → 로그인 후 계정 설정에서 비밀번호 변경. 단, 가입 시 이메일 등록이 전제. 미등록이면 자가 복구 불가, 관리자 문의 필요.

Q: 내보낸 캐릭터 JSON을 다른 곳에서 사용 가능?

가능합니다. 내보내기는 플랫폼 내부 형식의 완전한 레시피 + 콘텐츠 스냅샷이며, 본 플랫폼의 다른 계정에 직접 가져올 수 있습니다. 이론상 다른 도구 입력으로도 가능하지만 본 형식 지원이 필요합니다.

Q: 차단당했을 때 이의 신청 방법?

차단 레코드에 사유와 차단자가 표시됩니다. 신청 채널은 다음 절 참조.

15피드백

버그 발견, 제안, 운영 결정에 이의가 있을 때. 채널:

  • 사이트 내 토론: 관련 문서의 토론 페이지에 새 스레드 작성(콘텐츠 관련 문제용)
  • 신고: 위반 콘텐츠는 🚩 신고 버튼(구체적 콘텐츠 문제용)
  • 이메일 / 커뮤니티: 공개 전 관리자가 외부 연락처 보충
💡 좋은 피드백 작성 팁

포함하면 유용: 구체적 페이지 URL, 한 일, 예상 결과, 실제 결과, 브라우저와 OS. 이 정보가 담긴 1건이 "안 됨" 10건보다 훨씬 유용합니다.
